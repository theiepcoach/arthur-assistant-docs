## Orvared Evaluator Module Enhancement

**Context:** Working with MuleHair Labs on improving Orvared's evaluator module for special education evaluators.

### Current State
- Basic evaluation list: Student, Type, Status, Evaluator, Start/Due/Meeting dates, Notes
- Statuses: Not Started → In Progress → Lead Review → Finalized → Released

### Recommended Changes (Simplified)

**Add these fields to `evaluations` table:**

| Field | Purpose |
|-------|---------|
| `referral_source` | Parent/teacher/doctor referral source |
| `consent_received_date` | When consent obtained (starts the clock) |
| `suspected_disabilities` | What areas being eval'd (ED, SLD, Speech, etc.) |
| `coordinator_id` | Case owner (clearer than "evaluator") |
| Auto-calc: `days_remaining` | Countdown from consent_date + 45 school days |

### Simplified Status Workflow
- Not Started
- In Progress
- Complete

(removed Lead Review as overcomplication for v1)

### Key Insight
- Evaluators are coordinators, not doers
- The coordinator assigns tasks but doesn't do all assessment themselves
- Deadline tracking is 80% of the value
